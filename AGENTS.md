# Codex 维护说明

## 项目语言

本仓库文档和说明以中文为主。代码中的关键注释也可使用中文，以便大学生团队快速理解和维护。

## 代码要求

- 代码应保持简单、可读，优先服务 openvela AI 硬件开发者大赛 MVP 演示。
- 不要引入复杂依赖，不要为了短期演示增加难以维护的框架。
- Python 脚本优先保持无第三方依赖，确保基础 Python 3 环境可直接运行。
- 固件侧示例应围绕屏幕、按键、震动、喇叭、日志等核心硬件交互展开。

## 文档要求

- 文档要适合比赛提交，语气正式、清晰，突出目标用户、应用场景、技术路线和演示价值。
- 所有修改都要考虑 openvela AI 硬件开发者大赛场景，避免偏离无障碍 AI 沟通与伴行终端主题。
- 新增功能时同步更新 README、项目说明或相关目录文档。

## AI Coding 日志

每次新增代码或文档，都要同步更新 `ai_coding_logs/` 中的 AI Coding 日志，记录使用工具、完成内容、AI 参与比例和后续计划。

## 隐私与安全

- 默认不保存原始音频。
- 日志只记录必要事件、模式和用户确认后的文字内容。
- 涉及求助、定位、联系人通知等能力时，必须明确用户授权边界。
