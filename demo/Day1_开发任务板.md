# Day 1 开发任务板

## 今日总目标

进入 VelaBridge 三天 MVP 实做阶段，完成硬件交互、AI Demo、本地测试和演示素材的第一轮任务闭环，为第 2 天联调打基础。

## 队员 A：项目管理、AI Coding 日志、演示文案

- 维护 Day 1 任务进度，记录已完成、进行中和阻塞事项。
- 更新 AI Coding 日志，说明 ChatGPT / Codex 在任务拆解、文档和代码生成中的参与情况。
- 编写 1 到 2 分钟演示文案，控制表达清晰、正式、适合比赛展示。
- 汇总当天照片、录屏、硬件测试结果和问题清单，准备第 2 天复盘。

## 队员 B：openvela 环境、屏幕、按键、震动、语音播报

- 检查 openvela 开发环境是否能正常构建和烧录基础示例。
- 完成屏幕点亮测试，验证启动后显示 `VelaBridge MVP`。
- 完成按键 A、B、C 的短按和长按识别验证。
- 完成震动马达短震、连续震动测试。
- 确认喇叭或功放模块的语音播报方案，先跑通常用固定语句播报。

## 队员 C：AI Demo、本地运行、测试文本、摘要生成

- 运行 `ai_agent/velabridge_ai_demo.py`，确认本地 Python MVP 流程可演示。
- 准备用于字幕简化的测试文本，覆盖日常沟通、问路、求助等场景。
- 优化快捷回复推荐示例，让输出更短、更适合屏幕显示和语音播报。
- 使用样例日志生成沟通摘要，确认摘要不保存原始音频，只展示确认后的文字信息。
- 整理后续接入 GPT API 或本地小模型的接口预留思路。

## 队员 D：硬件清单、接线图、演示场景、外观草模

- 核对 `hardware/BOM.md` 中的模块到货情况和替代方案。
- 绘制屏幕、按键、震动马达、喇叭、电源的第一版接线图。
- 设计听障沟通、视障模式、紧急求助三个演示场景。
- 制作可穿戴外观草模，优先保证固定、可握持、可展示。
- 记录硬件风险，例如模块不兼容、供电不足、接口数量不足。

## 今日验收标准

- 至少完成屏幕、按键、震动、语音播报中 2 到 3 项单模块验证。
- AI Demo 能在本地运行，并能输出字幕简化、快捷回复、摘要和日志。
- 演示流程有可执行脚本，硬件接线和风险有记录。
- AI Coding 日志同步更新，便于比赛材料追溯。
