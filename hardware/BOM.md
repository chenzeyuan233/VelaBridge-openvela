# VelaBridge MVP 硬件清单

| 硬件 | 用途 | MVP 是否必须 | 状态 |
| -- | -- | -------- | -- |
| openvela 开发板 | 运行 openvela，连接屏幕、按键、震动马达和音频模块 | 是 | 待接入 |
| OLED/LCD 屏幕 | 显示实时字幕、模式状态和提示信息 | 是 | 待测试 |
| 麦克风模块 | 采集外界语音，为后续语音识别提供输入 | 是 | 待选型 |
| 小喇叭/功放模块 | 播放快捷回复、求助语音和盲人模式菜单 | 是 | 待测试 |
| 实体按键 x3 | 触发快捷回复、震动提醒、盲人模式和求助操作 | 是 | 待接线 |
| 震动马达模块 | 提供状态提醒、盲人模式反馈和紧急提示 | 是 | 待测试 |
| 电池模块 | 支持可穿戴设备移动演示 | 是 | 待选型 |
| 杜邦线/面包板 | 快速完成硬件连接和原型验证 | 是 | 准备中 |
| 可选：摄像头 | 后续用于环境识别、文字识别或辅助导航 | 否 | 第二阶段评估 |
| 可选：NFC 模块 | 后续用于场景卡片、身份辅助或快捷配置 | 否 | 第二阶段评估 |
| 可选：ToF/超声波模块 | 后续用于距离检测、避障提醒和空间辅助 | 否 | 第二阶段评估 |

## 备注

第一阶段 MVP 优先验证基础交互链路，硬件选型以易接线、易调试、稳定演示为主。第二阶段再根据佩戴形态、功耗和安全性进行结构优化。
