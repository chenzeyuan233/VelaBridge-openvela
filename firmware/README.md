# openvela 固件侧 MVP 说明

本目录用于存放 openvela 端固件代码、硬件驱动适配说明和联调记录。第一阶段目标是完成基础硬件交互闭环，而不是实现最终产品级固件。

## 第一阶段硬件交互

- 屏幕显示字幕：显示 AI 简化后的短字幕、当前模式和关键状态。
- 按键触发快捷回复：通过实体按键快速触发常用表达，降低用户操作成本。
- 震动马达提醒：通过不同震动模式反馈普通提醒、模式切换和紧急状态。
- 喇叭语音播报：播放快捷回复、求助内容和盲人模式语音菜单。
- 日志保存：记录用户操作、设备状态、字幕摘要和求助事件。

## MVP 交互逻辑

| 操作 | 设备行为 | 日志事件 |
| -- | -- | -- |
| 按键 A | 显示“请说慢一点，谢谢”并可触发语音播报 | quick_reply |
| 按键 B | 触发震动提醒，用于提示对方或确认状态 | vibration |
| 按键 C | 播报“我需要帮助” | help_voice |
| 长按按键 C | 进入盲人模式，播报语音菜单并启用震动反馈 | blind_mode_enter |
| 长按求助键 | 触发紧急求助，播报求助语音并记录日志 | emergency_help |

## 建议震动编码

- 短震 1 次：普通确认，例如按键已触发。
- 短震 2 次：模式切换，例如进入盲人模式。
- 长震 1 次：重要提醒，例如需要用户确认。
- 连续强震：紧急求助已触发。

## 日志字段建议

```json
{
  "timestamp": "2026-06-23T10:00:00+08:00",
  "event_type": "quick_reply",
  "content": "请说慢一点，谢谢",
  "mode": "normal"
}
```

## 后续计划

第一阶段完成屏幕、按键、震动、喇叭和日志链路后，第二阶段将继续接入真实语音识别、AI 摘要同步、低功耗策略和更多传感器扩展。
